Preprocessing the description Column

There are a few things we’ll want to do before breaking our descriptions into individual words. First, we’ll convert all letters to lowercase, so that neighborhood and Neighborhood are not counted as two distinct tokens.

zillow.loc[:, "description"] = zillow["description"].str.lower()

Notice that all of the characters in the description column are now lowercase! You may want to also remove punctuation marks and digits from the description column. You can do this with .str_replace() and a regular expression, but I’ll leave that to you! Now let’s tokenize our descriptions.

Tokenizing the description Column

We’ll need to split the description column across each whitespace character found in our descriptions. From there, we’ll need to add the priceRange category into our data frame of word tokens. Then we’ll use .melt() to transform our data frame from wide to long format. You can un-comment each tokens_df line in the code cell below to see the state of the data you are working with after each step.

tokens_df = zillow["description"].str.split(" ", expand = True)

tokenized_descriptions = tokens_df.melt(id_vars = ["priceRange"], value_name = "word").drop("variable", axis = 1).reset_index(drop = True)

Perhaps not surprising, the most frequent words are “low-information” words. They don’t really tell us about the properties – these words/tokens are commonly referred to as stopwords. I’ll read in a dictionary of common stopwords and then we can use what we know about Python merges (specifically anti-joins) in order to filter out the stopwords.

Removing Stopwords

As mentioned, stopwords are commonly occurring tokens that are unlikely to provide useful information for our analyses. There are several lists of stopwords available – I found one that I like here. We could even add to this list if we wanted (and we probably do, since we know that all of these descriptions have to do with homes). I’ll leave that to you as well though.

Let’s read in the data frame of stopwords.

stopwords = pd.read_csv("https://raw.githubusercontent.com/juliasilge/tidytext/main/data-raw/stop_words.csv")

We can see the first few stopwords below, ignoring the lexicon column.

Now let’s use an anti-join to remove those words!

temp_list_outer = tokenized_descriptions.merge(stopwords, how = "outer", indicator = True, left_on = "word", right_on = "word")

tokenized_no_stopwords = temp_list_outer[(temp_list_outer._merge == "left_only")].drop(["_merge"], axis = 1).reset_index()

That looks better! There are still some issues here with spaces, symbols, and special characters. We can remove those with regular expressions. I’ll leave most of this to you, but below I’ll do a bit of naive filtering to remove any tokens containing fewer than 5 characters and also any NULL tokens.

tokenized_no_stopwords = tokenized_no_stopwords[tokenized_no_stopwords["word"].notnull()]
tokenized_no_stopwords = tokenized_no_stopwords[tokenized_no_stopwords["word"].apply(lambda x: len(x) > 4)]

A Plot of Most Frequent Words in Descriptions

Let’s finish this off by producing a barplot of the most commonly occurring words in each priceRange category.

tokenized_description_counts = tokenized_no_stopwords.groupby(["priceRange", "word"]).size().reset_index()
tokenized_description_counts.columns = ["priceRange", "word", "count"]
tokenized_description_counts = tokenized_description_counts.sort_values("count", ascending = False)

(
  ggplot(data = tokenized_description_counts[tokenized_description_counts.groupby('priceRange')['count'].rank(ascending=False) <= 15]) +
  geom_col(aes(x = "word", y = "count", fill = "priceRange"), show_legend = False) + 
  coord_flip() +
  facet_wrap("~priceRange", scales = "free")
)
